<!-- ce projet et une realisation d'une application web de gestion des tavhes utilison MVC POO PHP


Application de gestion de tâches en ligne, souhaite améliorer l'expérience de ses utilisateurs en intégrant des fonctionnalités avancées en utilisant PHP orienté objet et l'architecture MVC. L'objectif est de permettre aux utilisateurs de gérer efficacement leurs tâches avec une interface utilisateur intuitive et des fonctionnalités robustes. Pour ce faire, l'application doit être développée en suivant les principes de la POO (Programmation Orientée Objet) et en implémentant une architecture MVC basique.

Objectifs d'apprentissage :
Maîtriser les concepts avancés de la POO pour structurer et organiser le code en PHP de manière efficace.
Comprendre l'architecture MVC et son application dans le développement d'applications web pour séparer la logique métier de la présentation.
User Stories
En tant qu'utilisateur, je dois pouvoir créer un compte et me connecter pour voir et accéder à mes tâches.
En tant qu'administrateur, je veux pouvoir ajouter, modifier, supprimer mes tâches.
En tant qu'utilisateur, je souhaite organiser mes tâches par catégories et priorités.
En tant qu'utilisateur, je veux pouvoir marquer mes tâches comme complètes ou incomplètes.
En tant qu'utilisateur, je souhaite rechercher des tâches spécifiques par mots-clés.
Design
Interface responsive qui s'adapte aux différents appareils (mobiles, tablettes, ordinateurs).
Utilisation de pictogrammes pour une navigation intuitive.
Palette de couleurs simple pour une interface claire et lisible.
Structure
Page d'accueil avec un aperçu des tâches récentes et une barre de recherche pour trouver des tâches spécifiques.
Section dédiée aux tâches avec des filtres par catégorie et priorité.
Page de détail de tâche avec des informations complètes, des options de modification et de suppression.
Fonctionnalité avec PHP OOP et MVC
Structure MVC : Diviser le code en trois composants principaux (Modèle, Vue, Contrôleur) pour une meilleure organisation et maintenance du code.
Héritage et Polymorphisme : Utiliser ces concepts pour créer des classes de base et des classes dérivées pour les utilisateurs et les tâches.
Encapsulation : Protéger les propriétés des classes en utilisant des méthodes getter et setter.
Espaces de noms : Organiser les classes et les interfaces dans des namespaces pour éviter les conflits de noms et améliorer la lisibilité du code. -->
